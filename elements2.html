Data Cleaning in Power Query

In this project, I transitioned from SQL to Power Query to clean and transform data. This section focuses on tasks like removing rows, changing headers, adjusting data types, and more, providing a detailed walkthrough of each process. The dataset I'm working with is a Food Prep dataset that contains a list of food items and their price from different stores, which I sourced from Alex the Analyst, a youtuber whose video helped my analytical skills.

Apocolypse Food Prep Data in Power Query

As we can see, the data looks very rough and disorganized, there are some blank columns and null values. The goal is to get rid of some of these and clean it up to make it more usable for power bi visualizations

Key Tasks

Some of the cleaning tasks i would be focusing on are ;

Removing Rows

The first thing i want to do is to remove the top rows of all nulls because we don't need them. They are completely useless in our dataset. I can do this with a quick button at the top of our power query ribbon

Changing Headers

After removing the top two rows, the next thing is to make sure we have the right column headers. Fields like "Location, Product" should be our main column headers. This can be changed by going to "Transfrom" and changing the first rows as headers.

Changing Data Types

Changing to the correct data type is a crucial data cleaning practice because it ensures that the data is processed and analyzed correctly. We have to make sure we convert to the right data type to ensure data integrity, data consistency and accurate calculations and comparison


"Date" data type changed from decimal number to fixed decimal number

Removing rows with x values

Now our data is looking more like it, but we still have some irrelvant fields/values such as "Costco total", "Walmart total". We don't really need these so we can get rid of them. I can do this by clicking on the dropdown menu next to Location and selcecting "Remove Empty".

Removing columns

In the grand scheme of things, i don't think i need the grand total in this data set (no pun intended), so i am going to remove the column


Removing the column "Grand total"

Now, our data looks good visually, everything looks well set up. A problem we have is our date columns dosen't really work too well with Power bi visualization so what we are going to need to do is to Pivot our date where they are actually going to be rows


Pivot/Unpivoting

Pivoting in Power Query involves transforming rows into columns. It's used when you want to summarize data by categorizing it into columns based on unique values in a specific field.
Unpivoting is the reverse of pivoting. It takes columns and turns them into rows, which is helpful when your data is spread across multiple columns but you want to consolidate it into a more analysis-friendly format. For example, if you have a table where the years are columns and the sales figures are in the cells, unpivoting would turn those year columns into rows, making the data more suitable for trend analysis which is exactly the case in our dataset.

First thing i'm going to do is to transform the date columns by selecting all the date cells and transforming it by selcecting "Unpivot Columns".


After Unpivoting, i did some extra cleaning to finalize our data before saving the transformation. I renamed Location to "Store", Attributes to "Dates", Values to "Product Cost" and changed the data type of Date to that of a date.
